L'algorithme du Minimax

Le gain d'une partie est la somme des pièces gagnées par le premier joueur à laquelle on soustrait la somme des pièces gagnées par le second.
Si le résultat est positif, le premier joueur est vainqueur.

Prouvez que, dans le jeu présenté ici, le gain ne peut jamais être nul. (solution)

Si on appelle MAX le premier joueur (qui cherche à maximiser le gain) et MIN le second, on en déduit l'algorithme suivant :

function max(values, F, L) {
    if (F == L) return values[F];
    return Math.max(
        values[F] + min(values, F + 1, L),
        values[L] + min(values, F, L - 1)
    );
}

function min(values, F, L) {
    if (F == L) return -values[F];
    return Math.min(
            - values[F] + max(values, F + 1, L),
            - values[L] + max(values, F, L - 1)
    );
}

Le paramètre values est un tableau des valeurs des pièces. Par exemple : [2,3,5,4,6,8,10,1,9,7]. F est l'indice de la première pièce (en partant de la gauche) et L est l'indice de la dernière (L = last).

Pour connaître le gain d'une partie qui commence par la prise de la première pièce (first), on écrit :

var gain = values[0] + min(values, 1, 9);

Et pour le gain résultant de la prise de la dernière pièce :

var gain = values[9] + min(values, 0, 8);

Pour ce jeu, l'algorithme est très rapide puisqu'il ne doit analyser que 92=512 parties possibles. Mais c'est le même genre d'algorithme (avec quelques optimisations comme le alpha-beta pruning) qui sert à l'intelligence artificielle d'un jeu d'échec.
